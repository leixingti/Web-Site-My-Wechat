## ⚡ 两种使用方式

### 方式1️⃣：本地测试（5分钟）

**适合：** 想先看看效果

1. **下载项目文件夹**
   - 下载 `wechat-rss-aggregator` 文件夹到您的电脑

2. **安装 Node.js**（如果还没有）
   - 访问：https://nodejs.org/
   - 下载并安装 LTS 版本

3. **打开终端**
   - Windows：按 `Win + R`，输入 `cmd`，回车
   - Mac：按 `Cmd + Space`，输入 `terminal`，回车

4. **进入项目文件夹**
   ```bash
   cd 您下载的路径/wechat-rss-aggregator
   ```

5. **安装依赖**
   ```bash
   npm install
   ```
   等待 30 秒左右...

6. **启动服务器**
   ```bash
   npm start
   ```

7. **打开浏览器**
   访问：http://localhost:3000

🎉 **您应该能看到网站界面了！**

### 方式2️⃣：部署到 Railway（推荐，30分钟）

**适合：** 想要一个在线网站，任何人都能访问

#### 第1步：上传到 GitHub（10分钟）

**1.1 注册 GitHub**
- 访问：https://github.com/
- 点击 "Sign up" 注册账号

**1.2 创建新仓库**
- 登录后，点击右上角 `+` → "New repository"
- Repository name：`wechat-rss-aggregator`
- 选择 "Public"
- 点击 "Create repository"

**1.3 上传文件（简单方法）**
- 在新建的仓库页面，点击 "uploading an existing file"
- 打开您电脑上的 `wechat-rss-aggregator` 文件夹
- **选择文件夹内的所有文件**（不要选文件夹本身）
- 拖拽到浏览器窗口中
- 滚动到底部，点击 "Commit changes"

⏱️ 等待上传完成（1-2分钟）

#### 第2步：部署到 Railway（15分钟）

**2.1 注册 Railway**
- 访问：https://railway.app/
- 点击 "Login"
- 选择 "Login with GitHub"（用您的GitHub账号登录）

**2.2 创建新项目**
- 登录后，点击 "New Project"
- 选择 "Deploy from GitHub repo"
- 授权 Railway 访问您的 GitHub
- 选择 `wechat-rss-aggregator` 仓库

**2.3 等待部署**
- Railway 会自动：
  - 检测到 Node.js 项目
  - 安装依赖
  - 启动服务器
- 等待 3-5 分钟，直到状态变为 "Active" ✅

**2.4 获取网站链接**
- 点击您的项目
- 点击 "Settings" 标签
- 找到 "Networking" → "Public Networking"
- 点击 "Generate Domain"
- 复制生成的链接（类似：https://xxx.railway.app）

**2.5 访问您的网站**
- 在浏览器中打开刚才复制的链接
- 🎉 **您的网站已经在线了！**

#### 第3步：配置RSS源（5分钟）

现在网站是运行了，但还没有真实的文章数据。您需要配置RSS源：

**3.1 返回 GitHub 仓库**
- 访问：https://github.com/您的用户名/wechat-rss-aggregator

**3.2 编辑 fetcher.js**
- 点击 `fetcher.js` 文件
- 点击右上角的铅笔图标（Edit）
- 找到第 14-21 行：
  ```javascript
  const RSS_FEEDS = [
    {
      name: '示例公众号1',
      url: 'https://rsshub.app/wechat/mp/articles/example1',
    },
    {
      name: '示例公众号2', 
      url: 'https://rsshub.app/wechat/mp/articles/example2',
    }
  ];
  ```

**3.3 替换为真实RSS源**

**获取RSS源的几种方法：**

**方法A：使用 RSSHub（推荐）**
- 访问：https://docs.rsshub.app/routes/new-media#wei-xin
- 查找微信公众号RSS生成方法
- 例如：`https://rsshub.app/wechat/mp/msgalbum/公众号ID/专辑ID`

**方法B：使用其他RSS服务**
- FeedDD：https://feeddd.org/
- WeRSS：https://werss.app/
- 其他RSS生成服务

**示例配置：**
```javascript
const RSS_FEEDS = [
  {
    name: '人民日报',
    url: 'https://rsshub.app/wechat/mp/msgalbum/rmrbwx/2667343089935278081',
  },
  {
    name: '央视新闻',
    url: 'https://rsshub.app/wechat/mp/msgalbum/cctvnewscenter/2650819981740425218',
  }
];
```

**3.4 保存更改**
- 滚动到底部
- 点击 "Commit changes"
- 点击 "Commit changes"（确认）

**3.5 等待重新部署**
- 返回 Railway 页面
- 会看到自动触发新的部署
- 等待 2-3 分钟

**3.6 刷新网站**
- 打开您的网站
- 等待几秒（初次抓取需要时间）
- 点击页面上的"刷新"按钮
- 🎉 **您应该能看到文章了！**

## 🔧 常见问题

### Q1: 网站显示"暂无文章"怎么办？

**原因：** RSS源配置不正确或还未抓取

**解决：**
1. 检查 `fetcher.js` 中的RSS链接是否正确
2. 点击网站上的"刷新"按钮手动触发抓取
3. 等待1-2分钟让定时任务运行

### Q2: Railway 部署失败怎么办？

**解决：**
1. 在 Railway 点击 "Deployments"
2. 点击失败的部署
3. 点击 "View Logs" 查看错误
4. 复制错误信息告诉我，我帮您解决

### Q3: 如何修改网站颜色？

编辑 `public/styles.css` 第 6 行：
```css
--primary-color: #2563eb;  /* 改为您喜欢的颜色 */
```

保存并推送到 GitHub，Railway 会自动更新。

### Q4: 如何更改抓取频率？

编辑 `server.js` 第 57 行：
```javascript
// 每小时：'0 * * * *'
// 每30分钟：'*/30 * * * *'
// 每天8点：'0 8 * * *'
cron.schedule('0 * * * *', ...
```
